最详细、可直接落地的：OpenProse 工作流 每 Step 事件上报 实现方案
我会用业务人员 + 前端都能看懂的方式，把「怎么改 Prose、后端怎么发、前端怎么收」讲透，你直接复制就能用，完全匹配你要的：
左侧步骤树高亮 + 中间窗口流式打字输出 + 每一步都可见。
一、核心原理（一句话讲清）
OpenProse 工作流 每执行一个 step，就通过 emit 内置指令 向 WebSocket 推送一条标准事件消息。
前端监听 WebSocket，收到一条消息就渲染一步，实现：
左侧步骤节点 实时高亮
中间窗口 流式打字输出
业务人员 全程可见执行过程
二、必须统一的「事件消息格式」（前后端通用）
这是整个分步输出的灵魂，所有 step 都用这套结构，前端不用改逻辑，直接解析。
json
{
  "workflowId": "task_20250520_123456",
  "stepId": "step_1_intent",
  "stageName": "阶段1｜用户意图识别",
  "eventType": "step_begin / token_stream / step_finish / workflow_end",
  "streamContent": "正在分析你问题的类型...",
  "fullContent": "{完整JSON结果}",
  "status": "running / success / error / skip"
}
4 种必须用到的事件（对应你界面行为）
step_begin
前端行为：左侧步骤 变黄高亮，中间窗口开始显示标题
token_stream
前端行为：中间窗口 逐字流式输出（DS 思考同款）
step_finish
前端行为：左侧步骤 变绿完成，缓存该步骤完整内容
workflow_end
前端行为：显示最终汇总报告
三、OpenProse 改造方案（逐行可复制）
我给你最标准、最通用的写法，OpenClaw 原生支持，不需要额外插件。
模板：每个 Step 固定三段式（开始 → 流式 → 结束）
prose
# ==========================================
# 【步骤 X】左侧显示的阶段名称
# ==========================================
emit: {
  "eventType": "step_begin",
  "stepId": "step_x_xxxx",
  "stageName": "阶段X｜XXX",
  "status": "running"
}

let 步骤变量 = session: 智能体
  stream: true                # 开启流式输出（必须）
  prompt: |
    这里写你的提示词...

  # 逐token推送流式内容（前端打字机）
  on stream delta:
    emit: {
      "eventType": "token_stream",
      "stepId": "step_x_xxxx",
      "streamContent": "${delta}"
    }

# 步骤结束推送完成事件
emit: {
  "eventType": "step_finish",
  "stepId": "step_x_xxxx",
  "stageName": "阶段X｜XXX",
  "fullContent": "${步骤变量}",
  "status": "success"
}

五、后端（OpenClaw）做了什么？（你不用关心，但我讲清楚）
你不需要写后端代码，OpenClaw 内部自动完成：
识别 Prose 里的 emit 指令
把 JSON 消息 扔进 WebSocket 队列
根据 workflowId 推送给 对应前端窗口
流式 token 会实时一条一条推，实现打字机效果
一句话：
你只需要在 Prose 里写 emit，后端自动转发到前端 WebSocket。
六、前端怎么接收与展示（最简代码）
你前端窗口用 WebSocket 监听下面消息，就能实现：
左侧步骤树 自动生成、自动高亮、自动完成
中间窗口 逐字流式输出
javascript
运行
const ws = new WebSocket("ws://你的openclaw地址/ws/workflow")

ws.onmessage = (evt) => {
  const msg = JSON.parse(evt.data)

  switch (msg.eventType) {
    case "step_begin":
      // 左侧步骤 → 变黄高亮
      addStepNode(msg.stepId, msg.stageName, "running")
      break

    case "token_stream":
      // 中间窗口 → 逐字追加（打字机效果）
      appendToCenterPanel(msg.streamContent)
      break

    case "step_finish":
      // 左侧步骤 → 变绿完成
      updateStepStatus(msg.stepId, "success")
      break

    case "workflow_end":
      showFinalReport(msg.fullContent)
  }
}
