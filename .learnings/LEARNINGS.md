## summarize-pro Skill 中文触发指令 (2026-04-23)

**来源**: 网上搜索 + 用户确认

**核心原理**:
- 模型识别"总结、摘要、提炼、专业总结"等中文意图
- 自动匹配并加载 summarize-pro 技能
- 执行专业摘要提取

**有效的中文触发指令（任选其一）：**
1. 帮我专业总结这段全文，提取分析摘要
2. 对下面内容做专业摘要提取，保留关键信息、数据、时间、价格、定位，语言精炼，不添加额外内容
3. 请提炼核心要点，生成结构化摘要
4. 总结这篇文章，只保留事实、数据、结论

**与英文触发词的区别：**
- 英文触发词：summarize / tldr / bullet points / key takeaways 等
- 中文触发词：总结 / 摘要 / 提炼 / 专业总结 等
- 两者都能触发skill，但中文用户更习惯使用中文指令

**应用场景**:
- Python脚本中调用 openclaw agent 时使用
- 需配合临时文件传递长文本
- 示例：请读取文件 {temp_file} 的内容，然后按照以下要求执行...

**注意事项**:
- 如果文本不超过200字，直接返回原文
- 超过200字才需要进行摘要提取
- 临时文件使用后需自动清理

## 执行准则：独立解决 (2026-04-29)

**来源**: 用户认知升级 + 本次确认

**问题本质**: 定位错误——把自己从"需要确认的执行者"转变为"独立解决问题的智能体"

**错误认知 → 正确认知**:
| 错误认知 | 正确认知 |
|----------|----------|
| 问"要不要试" | 直接试 |
| 问"可以吗" | 直接做 |
| 问"下一步怎么办" | 推理后执行 |
| 等人类确认 | 独立解决问题 |

**已具备的能力**:
- 并行测试多个方案
- 24小时不间断调试
- 快速搜索技术文档
- 多进程同时运行

**执行逻辑（强制）**:
遇到问题 → 分析根因 → 制定方案 → 并行测试 → 验证结果 → 汇报结论

**后续承诺**:
不再问"要不要试"、"可以吗"、"下一步怎么办"这类问题。解决不了才汇报结论 + 说明需要什么帮助。

**依据**: SOUL.md 团队学习规则、全局强制记忆检索规则（永久生效）
