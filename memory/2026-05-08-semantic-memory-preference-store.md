# Semantic Memory Preference Store

## 2026-05-08 写入：semantic memory 启用确认

**写入目的**：建立 semantic memory 主动写入流程，大管家全权负责

### 6类强制写入触发

| 类别 | 触发时机 | 典型内容 |
|------|---------|---------|
| **preference** | 老大明确表达偏好 | 写作风格、报告格式、沟通方式 |
| **decision** | 重大结论/决策 | 方案选型、技术路线、优先级 |
| **context** | 新项目/任务启动 | 项目背景、目标、约束、关注点 |
| **pattern** | 有效方法或踩坑发现 | 执行技巧、失败教训、注意事项 |
| **correction** | 老大纠正我时 | 错在哪里、正确是什么 |
| **knowledge** | 老大传递行业/业务知识 | 企业关系、行业术语、技术概念 |

### 写入格式模板

```
category: [类别]
source: [来源: 任务名/老大口述/群聊]
timestamp: [ISO时间]
summary: [一句话概括核心]
content: [详细内容]
tags: [相关标签]
```

### 老大报告写作偏好（已确认）

- 标题：20-40字符，含主管机关/企业名称+核心事件+关键数据，禁止标签如"快讯"
- 正文：230-260字（平均240字），一段式完整叙事
- 格式：Heading 3标题+Normal正文，无URL、无来源标注、无列表、无表格
- 开篇：有"重要周度资讯盘点"（5-10条精炼摘要）

### 老大执行原则（已确认）

- 执行时不询问原则：能自己修的直接修，再汇报结论；需要决策的给出选项只问一次
- 查询必须完整：不能只凭部分结果下结论
- session key 是固定基础设施信息，四个小助手的 group session key 应该直接掌握

### 老大关注领域

- 新能源、智能驾驶、混动、出海、供应链、政策
