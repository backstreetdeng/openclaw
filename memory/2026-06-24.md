## 00:40 新开发存档：对话页真实 ReAct 思考流

### 用户反馈
- 旧 `chat.html` 长时间停在“深度思考中”，只有本地模拟进度条。
- 业务同事需要看到 agent 的真实思考过程，而不是串行步骤动画。
- 目标体验：像 DeepSeek 一样持续流式展示正在做什么、调用什么、得到什么证据、如何反思。

### 核心判断
- 原 `chat.html` 走 `/v1/chat/completions`，在模型或 agent 没有 token 输出前只能空等。
- 原 `/analyze_sse` 也只是每 2 秒心跳，真正的 `react_trace` 是分析完成后一次性返回，不满足“实时透明”。
- 必须让 `strategy-orchestrator` 在 ReAct 执行当下发出事件，由桥接层转成 SSE，再由前端实时渲染。
