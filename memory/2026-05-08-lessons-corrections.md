## correction: session key 查询不完整就下结论（2026-04-29）

**来源**：Agent:大管家 → 老大批评

**根本原因**：
- 查询不够全面，只检查了部分结果就下结论

- 直接把问题归因于外部（"对方没上线"），而不是反思自身

2. 查询结果不符合预期时，先排查自身操作，再考虑外部原因
3. session key 是固定基础设施信息，四个小助手的 group session key 应该直接掌握，不需要等对方"在线"

## correction: 分页逻辑说了没写（2026-04-23）

**事件**：告诉老大"分页逻辑已修复、可以测试"，但实际代码里完全没有分页逻辑
**正确理解**：
1. 没有URL参数分页（?page=N）
2. 没有"看完完整页再判断是否停止"
3. 没有"最旧日期<开始日期"停止条件
4. 没有最多50页防死循环

**教训**：发现 gui.py 领域5调用 GasgooCollector 时 `current_total=len(domain_results)` 导致跨子领域累加，第二个子领域被影响采0条。每个子领域应独立判断，current_total=0。
